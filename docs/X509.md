# X.509 Certificate Browser Setup Guide

This guide explains how to import user certificates and the CA trust chain into your browser (Chrome) to enable client certificate authentication when accessing the Ingress endpoint.

## Overview

To authenticate with the Ingress using X.509 client certificates, you need to:

1. **Import the Root CA** into your browser's trusted root store
2. **Import user certificates** (PKCS12 format) into your browser's certificate store
3. **Access the Ingress** - the browser will automatically present the certificate

## Prerequisites

- Certificates have been generated using `./scripts/certs/setup-all-certs.sh`
- Root CA certificate: `k8s/certs/ca/root/ca-root.crt`
- User certificates: `k8s/certs/users/*.p12` (e.g., `jdoe.p12`, `asmith.p12`, etc.)

## Step 1: Import Root CA into Browser

The Root CA must be trusted by your browser so it can validate certificates signed by the Intermediate CA.

### Chrome (macOS)

1. **Open Keychain Access** (macOS manages certificates for Chrome):

   ```bash
   open /Applications/Utilities/Keychain\ Access.app
   ```

2. **Import Root CA**:
   - Go to **File → Import Items...**
   - Navigate to: `k8s/certs/ca/root/ca-root.crt`
   - Select the file and click **Open**

3. **Trust the Root CA**:
   - Find "Acme Root CA" in the **login** keychain
   - Double-click it to open the certificate details
   - Expand **Trust** section
   - Set **When using this certificate** to **Always Trust**
   - Close the window and enter your password when prompted

4. **Verify**:
   - The certificate should show "This root certificate is trusted" in Keychain Access

### Chrome (Linux)

1. **Install certificate in system trust store**:

   ```bash
   # Copy CA to system certificates directory
   sudo cp k8s/certs/ca/root/ca-root.crt /usr/local/share/ca-certificates/acme-root-ca.crt
   
   # Update CA certificates
   sudo update-ca-certificates
   ```

2. **Restart Chrome** for changes to take effect

### Chrome (Windows)

1. **Open Certificate Manager**:
   - Press `Win + R`, type `certmgr.msc`, press Enter

2. **Import Root CA**:
   - Navigate to **Trusted Root Certification Authorities → Certificates**
   - Right-click **Certificates** → **All Tasks → Import...**
   - Click **Next** → **Browse** → Select `k8s/certs/ca/root/ca-root.crt`
   - Click **Next** → Select **Place all certificates in the following store**
   - Ensure **Trusted Root Certification Authorities** is selected
   - Click **Next** → **Finish**
   - Click **Yes** to install the certificate

3. **Restart Chrome** for changes to take effect

## Step 2: Import User Certificates

User certificates are in PKCS12 format (`.p12` files) and contain both the certificate and private key.

### Chrome (macOS)

1. **Import via Keychain Access**:

   ```bash
   open k8s/certs/users/jdoe.p12
   ```

   Or manually:
   - Open **Keychain Access**
   - Go to **File → Import Items...**
   - Navigate to: `k8s/certs/users/jdoe.p12`
   - Enter password when prompted (certificates generated by the script have **no password** - just press Enter)
   - The certificate will be imported into the **login** keychain

2. **Verify**:
   - In Keychain Access, select **login** keychain
   - Filter by **My Certificates**
   - You should see the user certificate (e.g., "jdoe")

### Chrome (Linux)

1. **Import certificate**:

   ```bash
   # Import into Chrome's certificate store
   certutil -A -n "jdoe" -t "P,,," -i k8s/certs/users/jdoe.p12 -d sql:$HOME/.pki/nssdb
   ```

   If the NSS database doesn't exist:

   ```bash
   mkdir -p $HOME/.pki/nssdb
   certutil -N -d sql:$HOME/.pki/nssdb --empty-password
   ```

2. **Restart Chrome** for changes to take effect

### Chrome (Windows)

1. **Import via Certificate Manager**:
   - Press `Win + R`, type `certmgr.msc`, press Enter
   - Navigate to **Personal → Certificates**
   - Right-click **Certificates** → **All Tasks → Import...**
   - Click **Next** → **Browse** → Select `k8s/certs/users/jdoe.p12`
   - Enter password when prompted (certificates generated by the script have **no password** - leave blank)
   - Click **Next** → Select **Automatically select the certificate store based on the type of certificate**
   - Click **Next** → **Finish**

2. **Restart Chrome** for changes to take effect

## Step 3: Access the Ingress

Once certificates are imported:

1. **Navigate to the Ingress URL** (e.g., `https://your-ingress.example.com`)

2. **Chrome will automatically**:
   - Detect that client certificate authentication is required
   - Show a certificate selection dialog
   - Present available user certificates

3. **Select your certificate**:
   - Choose the user certificate you want to use (e.g., "jdoe")
   - Click **OK**

4. **Authentication**:
   - Chrome will send the selected certificate to the server
   - The Ingress will extract the Subject DN from the certificate
   - The application will authenticate you based on the DN

## Troubleshooting

### Certificate Not Appearing in Selection Dialog

**Issue**: Chrome doesn't show your certificate when accessing the Ingress.

**Solutions**:

1. **Verify Root CA is trusted**:
   - Check that the Root CA is in the trusted root store
   - Ensure it's set to "Always Trust" (macOS) or in Trusted Root store (Windows/Linux)

2. **Verify certificate chain**:

   ```bash
   # Check certificate details
   openssl pkcs12 -in k8s/certs/users/jdoe.p12 -nokeys -clcerts -passin pass: | openssl x509 -text -noout
   ```

   - Verify the certificate shows the correct Subject DN
   - Check the Issuer matches "Acme Intermediate CA"

3. **Check certificate validity**:

   ```bash
   openssl pkcs12 -in k8s/certs/users/jdoe.p12 -nokeys -clcerts -passin pass: | openssl x509 -dates -noout
   ```

   - Ensure the certificate is not expired

### "Your connection is not private" Error

**Issue**: Chrome shows a security warning even after importing the Root CA.

**Solutions**:

1. **Clear browser cache and restart Chrome**
2. **Verify Root CA is in the correct store** (Trusted Root Certification Authorities)
3. **Check certificate chain** - ensure Intermediate CA is also trusted if required

### Certificate Selection Dialog Not Appearing

**Issue**: Chrome doesn't prompt for certificate selection.

**Solutions**:

1. **Check Ingress configuration** - ensure it's configured to require client certificates
2. **Verify user certificates are imported** - check Keychain Access (macOS) or Certificate Manager (Windows)
3. **Clear browser cache and restart Chrome**
4. **Try incognito mode** - sometimes helps with certificate selection

### Wrong Certificate Selected

**Issue**: Chrome selects the wrong certificate automatically.

**Solutions**:

1. **Remove other certificates** from your certificate store that might be auto-selected
2. **Use certificate selection dialog** - Chrome should show a dialog if multiple certificates match
3. **Check certificate Subject DN** - ensure it matches what the Ingress expects

## Verifying Certificate Installation

### macOS

```bash
# List certificates in login keychain
security find-identity -v -p ssl-client

# View specific certificate
security find-certificate -c "jdoe" -p
```

### Linux

```bash
# List certificates in NSS database
certutil -L -d sql:$HOME/.pki/nssdb
```

### Windows

```powershell
# List certificates in Personal store
Get-ChildItem -Path Cert:\CurrentUser\My
```

## Certificate Details

To view certificate information:

```bash
# View Subject DN
openssl pkcs12 -in k8s/certs/users/jdoe.p12 -nokeys -clcerts -passin pass: | openssl x509 -subject -noout

# View Issuer DN
openssl pkcs12 -in k8s/certs/users/jdoe.p12 -nokeys -clcerts -passin pass: | openssl x509 -issuer -noout

# View full certificate details
openssl pkcs12 -in k8s/certs/users/jdoe.p12 -nokeys -clcerts -passin pass: | openssl x509 -text -noout
```

## Security Notes

1. **Private Keys**: User certificates contain private keys. Keep `.p12` files secure and don't share them.

2. **Root CA**: The Root CA private key should be kept extremely secure. If compromised, all certificates become untrustworthy.

3. **Certificate Expiration**: User certificates are valid for 1 year. Regenerate them before expiration.

4. **Browser Storage**: Certificates are stored in the browser's certificate store. On shared machines, use separate user accounts.

5. **Revocation**: If a certificate needs to be revoked, remove it from the browser's certificate store and regenerate a new one.
